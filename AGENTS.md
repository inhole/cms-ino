# 작업 공통 규칙

모든 코드 수정 작업을 시작하기 전에 반드시 `mds/작업전_인코딩_체크리스트.md`를 먼저 읽고 그 절차를 따른다.

## 인코딩 규칙
- 텍스트 파일(`.java`, `.html`, `.md`, `.yml`, `.properties`)은 UTF-8(가능하면 BOM 없음)으로 저장한다.
- PowerShell에서 파일을 덮어쓸 때 기본 인코딩 저장을 사용하지 않는다.
- 한글이 포함된 파일은 수정 후 깨짐 여부를 확인한다.

## 검증 규칙
- 화면 템플릿 수정 후 닫는 태그 손상(`??/a>`, `/title>` 등) 여부를 검색해 확인한다.
- 컴파일 또는 테스트로 최소 1회 이상 검증한다.
