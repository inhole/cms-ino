# 작업 전 인코딩 체크리스트

## 1) 작업 시작 전
- 현재 터미널에서 한글이 깨져 보일 수 있으므로, 파일 저장 인코딩을 UTF-8로 강제한다.
- 기존 파일을 읽을 때 한글이 깨져 보이면 즉시 수정하지 말고 원문 손상 여부를 먼저 확인한다.

## 2) 파일 수정 시 규칙
- `.java`, `.html`, `.md`는 UTF-8(권장: BOM 없음)으로 저장한다.
- PowerShell 기본 `Set-Content` 저장을 피하고, 필요 시 UTF-8 명시 저장을 사용한다.
- 대량 치환/정규식 삽입 작업 후 HTML 태그가 깨졌는지 확인한다.

## 3) 수정 직후 필수 점검
- 아래 패턴이 남아 있는지 검색:
  - `??/a>`
  - `/title>`
  - `/h1>`
  - `/p>`
  - `硫`, `寃`, `愿` (대표적인 한글 깨짐 패턴)
- 깨진 문자열이 있으면 해당 파일을 전체 재작성하여 정리한다.

## 4) 최종 검증
- `./gradlew.bat compileJava`
- 필요한 경우 대상 기능 테스트 실행
- 화면 템플릿 경로와 라우트 경로가 일치하는지 확인
