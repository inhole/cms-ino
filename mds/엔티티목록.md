# CMS 엔티티 도메인 목록

## 생성된 엔티티 (총 23개)

### 1. 공통 엔티티
- **BaseEntity**: 모든 엔티티의 공통 필드 (등록일시, 등록자, 수정일시, 수정자)

### 2. 도메인 관리
- **Domain**: 도메인 정보 관리 (TB_DOMAIN)

### 3. 회원 및 권한 관리
- **Member**: 회원 정보 (TB_MEMBER)
- **Group**: 회원 분류/그룹 (TB_GROUP)
- **GroupAuth**: 그룹별 권한 설정 (TB_GROUP_AUTH)
- **MemberGrpRel**: 회원-그룹 관계 (TB_MEMBER_GRP_REL)
- **DeptInfo**: 부서/학과 정보 (TB_DEPT_INFO)

### 4. 메뉴 관리
- **Menu**: 사용자 메뉴 관리 (TB_MENU)
  - GNB/LNB 메뉴 구조
  - 페이지/게시판 연결
  - 다국어 지원

### 5. 메인 페이지 관리
- **Main**: 메인 페이지 설정 (TB_MAIN)

### 6. 페이지 관리
- **Page**: 정적 페이지 관리 (TB_PAGE)
  - 페이지 콘텐츠
  - SEO 설정
  - 템플릿 연결

### 7. 게시판 관리
- **Bbs**: 게시판 설정 (TB_BBS)
  - 게시판 타입, 권한, 템플릿
  - 파일첨부, 댓글, 카테고리 설정
- **BbsCategory**: 게시판 카테고리 (TB_BBS_CATEGORY)
- **BbsMaster**: 게시글 본문 (TB_BBS_MASTER)
- **BbsReply**: 게시글 댓글 (TB_BBS_REPLY)
- **BbsField**: 게시판 동적 필드 설정 (TB_BBS_FIELD)
- **BbsMasterField**: 게시글 동적 필드 값 (TB_BBS_MASTER_FIELD)

### 8. 배너 관리
- **Banner**: 배너 그룹 (TB_BANNER)
- **BannerSub**: 배너 상세 (TB_BANNER_SUB)
  - PC/모바일 이미지
  - 링크 설정
  - 모션 효과

### 9. 팝업 관리
- **Popup**: 팝업 관리 (TB_POPUP)
  - 노출 기간 설정
  - 이미지 업로드

### 10. 템플릿 관리
- **Template**: 페이지/게시판 템플릿 (TB_TEMPLATE)

### 11. 파일 관리
- **CommonFile**: 공통 파일 관리 (TB_COMMON_FILE)
- **CommonFileCode**: 파일 타입 코드 (TB_COMMON_FILE_CODE)

### 12. 코드 관리
- **Code**: 공통 코드 (TB_CODE)
- **CodeDetail**: 공통 코드 상세 (TB_CODE_DETAIL)

### 13. 시스템 관리
- **Ids**: 채번 관리 (TB_IDS)
- **Log**: 사용자 로그 (TB_LOG)

## 엔티티 관계도

```
Domain (도메인)
 ├─ Main (메인 페이지)
 ├─ Menu (메뉴)
 │   ├─ Page (페이지)
 │   └─ Bbs (게시판)
 │       ├─ BbsCategory (카테고리)
 │       ├─ BbsMaster (게시글)
 │       │   ├─ BbsReply (댓글)
 │       │   └─ BbsMasterField (동적 필드 값)
 │       └─ BbsField (동적 필드 설정)
 ├─ Popup (팝업)
 └─ GroupAuth (권한)
     └─ Group (그룹)
         └─ MemberGrpRel (회원-그룹 관계)
             └─ Member (회원)

Banner (배너)
 └─ BannerSub (배너 상세)

Template (템플릿)
 ├─ Page (페이지에서 사용)
 └─ Bbs (게시판에서 사용)

Code (공통코드)
 └─ CodeDetail (코드 상세)

CommonFileCode (파일 타입)
 └─ CommonFile (파일)
```

## 주요 특징

### 1. JPA Auditing 적용
- BaseEntity를 상속받아 자동으로 등록일시, 수정일시 관리
- `@EnableJpaAuditing` 활성화됨

### 2. IDENTITY 전략으로 PK 자동 생성
- **모든 엔티티의 PK가 `@GeneratedValue(strategy = GenerationType.IDENTITY)` 사용**
- 데이터베이스의 AUTO_INCREMENT 기능 활용
- TB_IDS 채번 테이블 불필요 → 삭제됨
- 타입: 대부분 `Long`, 파일 관련은 `Integer`

### 3. 복합키(Composite Key) 사용
- Menu (menuId IDENTITY + domainId)
- BbsMaster (bbsId + nttNo IDENTITY)
- BbsReply (bbsId + nttNo + rplNo IDENTITY)
- BbsField (bbsId + fieldCode)
- BbsMasterField (bbsId + nttNo + fieldCode)
- GroupAuth (grpId + domainId)
- MemberGrpRel (userId + grpId)
- CodeDetail (codeId + code)
- BannerSub (bannerId + subNo IDENTITY)

### 4. 양방향 관계 설정
- Lazy Loading 적용
- 외래키 관계를 통한 연관 엔티티 참조

### 5. Lombok 활용
- @Getter, @Setter, @Builder, @NoArgsConstructor, @AllArgsConstructor
- 코드 간소화 및 유지보수성 향상
